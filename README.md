# Yandex Meeting Sorter

Docker-сервис для Coldex: читает новые письма из Яндекс Почты, распознаёт тип совещания по теме письма, забирает вложенный TXT/MD-файл и загружает его в соответствующую папку Яндекс Диска.

## Поддерживаемые темы и папки

| Фрагмент темы письма | Папка на Диске | Имя файла |
|---|---|---|
| Совещание Техническое | Совещание Техническое | `YYYY-MM-DD_Совещание Техническое.txt` |
| Совещание по Продажам | Совещание по Продажам | `YYYY-MM-DD_Совещание по Продажам.txt` |
| Совещание по Одоо | Совещание по Одоо | `YYYY-MM-DD_Совещание по Одоо.txt` |
| Совещание по Маркетингу | Совещание по Маркетингу | `YYYY-MM-DD_Совещание по Маркетингу.txt` |
| Совещание по БП Реализация | Совещание по БП Реализация | `YYYY-MM-DD_Совещание по БП Реализация.txt` |
| Совещание по БП Продажи | Совещание по БП Продажи | `YYYY-MM-DD_Совещание по БП Продажи.txt` |
| Совещание по Лиду 5316 | Совещания по Лидам | `YYYY-MM-DD_Совещание по Лиду 5316.txt` |

Тема может содержать дополнительный текст, например `Конспект встречи: Совещание по Одоо`. Поиск выполняется по фрагменту темы без учёта регистра.

Если в один день придут два одинаково названных совещания, второе имя получит суффикс `_2`, затем `_3` и т. д.

## 1. Подготовка

Нужно иметь:

1. OAuth-токен с правом записи на Яндекс Диск.
2. Отдельный пароль приложения Яндекс Почты для IMAP.
3. Docker Engine и Docker Compose Plugin на сервере.

Основной пароль корпоративного Яндекс-аккаунта в контейнер не помещайте.

## 2. Настройка

```bash
cp .env.example .env
nano .env
```

Заполните как минимум:

```env
YANDEX_EMAIL=meetings@your-company.fi
YANDEX_APP_PASSWORD=...
YANDEX_DISK_TOKEN=...
```

Права на файл с секретами:

```bash
chmod 600 .env
```

### Первый запуск и старые письма

По умолчанию:

```env
PROCESS_EXISTING=false
```

Сервис запомнит последнее существующее письмо и начнёт работать только с новыми письмами.

Для теста на уже полученном письме задайте перед первым запуском:

```env
PROCESS_EXISTING=true
INITIAL_LOOKBACK_MESSAGES=100
```

После успешного теста можно вернуть `PROCESS_EXISTING=false`. Уже обработанные сообщения повторно не загрузятся благодаря SQLite-базе в `data/`.

## 3. Проверка конфигурации

```bash
docker compose build
docker compose run --rm meeting-sorter python -m app.main --check-config
```

Ожидаемый ответ:

```text
Configuration is valid
```

## 4. Однократный тест подключения и обработки

```bash
docker compose run --rm meeting-sorter python -m app.main --once
```

Посмотреть журнал:

```bash
docker compose logs --tail=200 meeting-sorter
```

## 5. Постоянный запуск

```bash
docker compose up -d --build
```

Проверка:

```bash
docker compose ps
docker compose logs -f meeting-sorter
```

Остановка:

```bash
docker compose down
```

Обновление после изменения кода:

```bash
docker compose up -d --build
```

Изменения только в `config/rules.yaml` применятся при перезапуске:

```bash
docker compose restart meeting-sorter
```

## 6. Что создаётся на Яндекс Диске

При `YANDEX_DISK_ROOT=/Совещания` сервис создаст:

```text
/Совещания/
├── Совещание Техническое/
├── Совещание по Продажам/
├── Совещание по Одоо/
├── Совещание по Маркетингу/
├── Совещание по БП Реализация/
├── Совещание по БП Продажи/
└── Совещания по Лидам/
```

Папки создаются по мере поступления соответствующих писем.

## 7. Что считается расшифровкой

Сервис обрабатывает вложения с расширением:

- `.txt`
- `.md`

Текст приводится к UTF-8, а на Диск всегда сохраняется как `.txt`.

Письма с известной темой, но без текстового вложения, записываются в базу со статусом `ignored_no_text_attachment`. Неизвестные темы получают статус `ignored_unknown_subject`.

## 8. Защита от дублей

Состояние находится в:

```text
data/meeting_sorter.sqlite3
```

В базе сохраняются UID письма, Message-ID, тема, статус и итоговый путь на Диске. Не удаляйте папку `data`, если не хотите повторной обработки сообщений.

Резервная копия:

```bash
tar -czf meeting-sorter-data-$(date +%F).tar.gz data/
```

## 9. Изменение правил

Правила находятся в:

```text
config/rules.yaml
```

Для фиксированной темы:

```yaml
- pattern: 'Совещание\\s+по\\s+Одоо'
  canonical_name: 'Совещание по Одоо'
  folder: 'Совещание по Одоо'
```

Для темы с изменяющимся номером используется регулярное выражение и `preserve_match: true`:

```yaml
- pattern: 'Совещание\\s+по\\s+Лиду\\s+\\d+'
  canonical_name: 'Совещание по Лиду'
  folder: 'Совещания по Лидам'
  preserve_match: true
```
