# Yandex Meeting Sorter

Docker-сервис для Coldex: читает письма из Яндекс Почты, извлекает название совещания из кавычек в теме письма, определяет папку и загружает вложенный TXT/MD-файл на Яндекс Диск без изменения имени и содержимого.

## Как распознаётся тема

Ожидаемый шаблон темы:

```text
Конспект встречи «Техническое совещание» от 14.07.2026;
```

Сервис использует только текст внутри кавычек:

```text
Техническое совещание
```

Текст до и после кавычек, включая дату, игнорируется. Сравнение выполняется без учёта регистра. Поддерживаются кавычки `«...»` и `"..."`.

Например, следующие варианты равнозначны:

```text
Конспект встречи «Техническое совещание» от 14.07.2026;
Конспект встречи «техническое совещание» от 14.07.2026;
Конспект встречи "ТЕХНИЧЕСКОЕ СОВЕЩАНИЕ" от 14.07.2026;
```

## Поддерживаемые названия и папки

| Название внутри кавычек | Папка на Диске |
|---|---|
| Техническое совещание / Совещание Техническое | Совещание Техническое |
| Совещание по Продажам | Совещание по Продажам |
| Совещание по Одоо / Совещание по Odoo | Совещание по Одоо |
| Совещание по Маркетингу | Совещание по Маркетингу |
| Совещание по БП Реализация | Совещание по БП Реализация |
| Совещание по БП Продажи | Совещание по БП Продажи |
| Совещание по Лиду 5316 | Совещания по Лидам |
| Совещание по Лиду №5316 | Совещания по Лидам |
| Совещание по Лиду #5316 | Совещания по Лидам |

## Что происходит с вложением

Исходное вложение сохраняется без изменений:

```text
2026-07-14 14:18 (MSK) Техническое совещание.txt
```

Результат:

```text
/Совещания/Совещание Техническое/2026-07-14 14:18 (MSK) Техническое совещание.txt
```

Сервис не меняет:

- имя файла;
- содержимое файла;
- кодировку;
- расширение `.txt` или `.md`.

Если файл с таким же именем уже существует в целевой папке, повторная загрузка пропускается.

## 1. Настройка

Создайте `.env`:

```bash
cp .env.example .env
```

Заполните:

```env
YANDEX_EMAIL=meetings@your-company.fi
YANDEX_APP_PASSWORD=...
YANDEX_DISK_TOKEN=...
YANDEX_DISK_ROOT=/Совещания
```

Для IMAP нужен отдельный пароль приложения Яндекс Почты. Основной пароль аккаунта использовать не следует.

## 2. Сборка и проверка

```bash
docker compose build --no-cache meeting-sorter
docker compose run --rm meeting-sorter python -m app.main --check-config
```

Ожидаемый ответ:

```text
Configuration is valid
```

Проверить список команд:

```bash
docker compose run --rm meeting-sorter python -m app.main --help
```

В списке должны присутствовать:

```text
--once
--check-config
--retry-ignored
--scan-last N
```

## 3. Проверить последние 100 писем

Команда явно просматривает последние 100 писем в `INBOX`, независимо от сохранённого `last_uid`:

```bash
docker compose run --rm meeting-sorter python -m app.main --scan-last 100
```

Это последние 100 писем почтового ящика, а не последние 100 совещаний. Уже успешно загруженные письма пропускаются. Ранее нераспознанные письма проверяются повторно.

`INITIAL_LOOKBACK_MESSAGES=100` применяется только при первом запуске с пустой SQLite-базой и `PROCESS_EXISTING=true`. Для уже работающей установки используйте `--scan-last 100`.

## 4. Повторить только ранее нераспознанные темы

```bash
docker compose run --rm meeting-sorter python -m app.main --retry-ignored
```

Команда повторно проверяет записи со статусом `ignored_unknown_subject`.

## 5. Однократная проверка новых писем

```bash
docker compose run --rm meeting-sorter python -m app.main --once
```

## 6. Постоянный запуск

```bash
docker compose up -d
docker compose logs -f meeting-sorter
```

Остановка:

```bash
docker compose down
```

После изменения Python-кода контейнер необходимо пересобрать:

```bash
docker compose down
docker compose build --no-cache meeting-sorter
docker compose up -d
```

## 7. Папки на Яндекс Диске

При `YANDEX_DISK_ROOT=/Совещания` используются папки:

```text
/Совещания/
├── Совещание Техническое/
├── Совещание по Продажам/
├── Совещание по Одоо/
├── Совещание по Маркетингу/
├── Совещание по БП Реализация/
├── Совещание по БП Продажи/
└── Совещания по Лидам/
```

## 8. Защита от дублей

Состояние хранится в:

```text
data/meeting_sorter.sqlite3
```

В базе сохраняются UID письма, Message-ID, тема, статус и пути загруженных файлов. При обновлении проекта не удаляйте `.env` и папку `data`.

## 9. Изменение правил

Файл:

```text
config/rules.yaml
```

Пример правила:

```yaml
- pattern: '(?:Совещание\s+Техническое|Техническое\s+совещание)'
  canonical_name: 'Совещание Техническое'
  folder: 'Совещание Техническое'
```

Регулярные выражения компилируются с `re.IGNORECASE`, поэтому регистр не учитывается.
